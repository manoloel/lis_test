## Проблемные места:

- Отсутствие обработки ошибок 
- Класс ASTM нагроможден низкоуровневой и высокоуровневой логикой 
- Проблемы с неймингом параметров и функций (большое количество сокращений ухудшающие читаемость) 
- Проблемы с тайпингом (Где-то ожидается строка, а приходит число) 
- Константы просто прописаны в коде 
 
## Решения:

- Добавить обработку ошибок (Здесь как будто не нужно аргументация), 
- Разделить класс ASTM: (Упростит поддержку и доработку модуля разделив уровни абстракции,  
сделает код более читаемым и менее связанным):
    - Выделить логику построения фреймов в отдельный класс
    - Выделить логику подключения и отправки данных в отдельный класс, сделать из него контекстный менеджер (Контекстный менеджер позволит в любом случае корректно закрывать соединения) 
- Избавиться от сокращений названий в коде (Улучшиться читаемость кода что позволит находить ошибки быстрее) 
- Выделить константы отдельно (Позволит легко изменить использование параметра во всем в случае изменения какого-либо из параметров) 
- В целом, наверное еще стоит добавить логгирование. Это позволит дебажить проблемы на уже запущенных устройствах, но тут нужно понимать какими ресурсами располагает устройство.
